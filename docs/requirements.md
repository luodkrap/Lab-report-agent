# 기계공학실험 레포트 에이전트 요구사항

## 목표

기계공학실험(2) 수업의 실험 레포트 작성을 지원하는 멀티 에이전트 AI 시스템을 구축한다.
실험 매뉴얼, 강의안, 레포트 양식, 실험 결과값, 족보(선배 레포트) 등 다양한 형식의 파일을 입력받아,
각 문항에 대한 고품질 한국어 공학 보고서 답변을 Word(.docx) 또는 즉시 복붙 가능한 형태로 생성한다.
AI 탐지기를 통과할 수 있는 자연스러운 문체를 유지하는 것이 최우선 요구사항이다.

## 사용자 역할

- 팀원 (학부생 4학년): 입력 파일을 제공하고, 생성된 답변을 각자 담당 문항에 맞게 활용한다. 사진 추천 결과를 참고하여 직접 삽입하고, 참고문헌 목록을 레포트에 붙여넣는다.

## 입력 파일 유형

| 파일 유형 | 내용 | 처리 방식 |
|-----------|------|-----------|
| PDF / Word(.docx) | 실험 매뉴얼, 레포트 양식 및 문항 | 직접 파싱 |
| pptx (강의안) | 관련 이론 슬라이드 | 사용자가 PDF로 변환 후 제출 (LibreOffice 또는 PowerPoint 내보내기 권장) |
| xlsx (실험 결과 데이터) | 측정값, 계산 결과 테이블 | openpyxl 기반 스크립트로 자동 텍스트 추출 |
| 이미지 (JPG, PNG 등) | 실험 결과 사진, 그래프, 측정 데이터 캡처 | 직접 파싱 |
| PDF / Word / 이미지 | 족보 (선배들이 작성한 이전 레포트 — 문체 참고 전용) | 직접 파싱 |

## 레포트 구성

- 과목: 기계공학실험(2)
- 실험 분야: 열전달, 기계진동, 유체역학, 동역학 (각 2개 실험, 총 8개)
- 레포트 종류: pre-lab, post-lab (실험당 각 1개, 총 16개)
- 팀 구성: 4명이 문항을 배분하여 분업 후 취합

## 에이전트 역할 정의

### Orchestrator
- 사용자로부터 입력 파일 및 레포트 종류(pre/post, 실험명)를 수신한다
- 전체 워크플로우를 조율하고 에이전트 간 작업을 순서에 맞게 분배한다
- xlsx 파일이 포함된 경우 Input Analyzer 실행 전에 xlsx 추출 스크립트를 먼저 실행하도록 지시한다
- 최종 결과물을 취합하여 출력 파일(.docx 또는 Markdown)로 저장한다

### Input Analyzer
- PDF, Word, 이미지 등 다양한 형식의 파일을 파싱한다
- 실험 매뉴얼에서 실험 목적, 원리, 절차를 추출한다
- 강의안(PDF로 변환된 pptx)에서 관련 이론 및 수식, 그림 목록을 추출한다
- 레포트 양식에서 문항 목록과 배점 구조를 파싱한다
- xlsx에서 추출된 텍스트 파일을 읽어 실험 결과값(수치, 계산값)을 정리한다
- 족보 파일에서 문체, 어투, 문장 구조, 표현 패턴만 추출한다 (내용은 추출하지 않음)

### Style Analyzer (족보 문체 분석 전담 에이전트)
- 족보(선배 레포트)의 문체 특성을 분석한다
- 분석 항목: 문장 길이 분포, 어미 패턴, 전문 용어 사용 방식, 수식 서술 방식, 단락 구성 방식
- 분석 결과를 "문체 가이드"로 정리하여 Humanizer에 전달한다
- 내용(수치, 분석 결론, 실험 결과 해석 등)은 일절 참고하지 않는다

### Theory Researcher
- Input Analyzer가 추출한 실험 주제를 바탕으로 관련 이론을 정리한다
- 열전달, 기계진동, 유체역학, 동역학 각 분야의 핵심 개념과 수식을 기술한다
- 강의안 내용을 기반으로 하되, 필요시 추가 배경지식을 보충한다
- 수식은 Word 수식 편집기에 붙여넣기 가능한 LaTeX 표기법으로 작성한다

### Report Writer
- 문항별로 답변 초안을 작성한다
- pre-lab: 실험 전 이론 이해, 예상 결과, 실험 방법 기술
- post-lab: 실험 결과 분석, 오차 원인, 이론과의 비교, 결론 기술
- 실험 결과값을 수치적으로 분석하여 답변에 반영한다
- 족보의 내용(결론, 수치, 분석)은 절대 참조하지 않는다
- 수식은 Word 수식 편집기에 붙여넣기 가능한 LaTeX 표기법으로 작성한다

### Humanizer
- Report Writer의 초안을 4학년 학부생이 직접 쓴 것처럼 자연스럽게 변환한다
- Style Analyzer가 정리한 문체 가이드를 적용한다
- AI 특유의 반복적 표현, 과도하게 매끄러운 문장 구조를 깨뜨린다
- 전문적이되 딱딱하지 않은 한국어 공학 보고서 스타일을 유지한다
- GPTZero, Turnitin AI 탐지기 기준을 통과할 수 있도록 문체를 조정한다
- 표절률이 낮게 유지되도록 원문 표현을 충분히 재구성한다

### Reviewer
- 답변 내용의 사실 정확성을 검토한다 (수식, 단위, 수치 등)
- 논리적 흐름과 문항 요구사항 충족 여부를 확인한다
- 미흡한 부분에 대해 구체적인 피드백을 작성하고 Report Writer 및 Humanizer에 재작업을 요청한다
- 족보 내용과의 유사도가 높은 표현이 있는지 확인하고, 발견 시 재작업을 요청한다
- 최종 품질 승인 후 Orchestrator에 결과를 반환한다

### Image Recommender
- 각 문항 및 단락에 삽입하기 적합한 사진/그림 유형을 추천한다
- 강의안이나 매뉴얼에 포함된 그림은 파일명 또는 페이지를 명시하여 추천한다
- 외부 이미지가 필요한 경우 검색 키워드, 추천 출처(교재, 논문, 위키피디아 등), URL을 제공한다
- 실제 삽입은 사용자가 직접 수행하며, 에이전트는 위치와 설명만 제안한다

### Reference Manager
- 사용된 강의안, 실험 매뉴얼, 교재 등을 참고문헌 형식으로 정리한다
- 추가 참고문헌이 필요한 경우 관련 논문, 교재, 웹 자료를 제안한다
- 참고문헌은 한국기계학회 또는 일반 공학 보고서 인용 형식에 맞게 작성한다

## 출력 결과물

### 주 출력물
- 문항별 답변이 포함된 레포트 Word(.docx) 파일 또는 즉시 복붙 가능한 Markdown 텍스트
- 수식은 Word 수식 편집기에 붙여넣기 가능한 LaTeX 형식으로 표기 (예: `$$Q = \dot{m} c_p \Delta T$$`)

### 부가 출력물
- 사진/그림 삽입 추천 목록 (삽입 위치, 설명, 출처)
- 참고문헌 목록 (인용 형식 준수)

## 제약사항

- AI 탐지기(GPTZero, Turnitin)를 통과해야 한다 — Humanizer의 핵심 책임
- 표절률을 최소화해야 한다 (목표: 20% 미만)
- 족보(선배 레포트)의 내용은 절대 참조하지 않는다 — 문체(어투, 문장 구조, 표현 패턴)만 참고 허용
- 모든 답변은 한국어로 작성한다
- 전문적이되 딱딱하지 않은 4학년 학부생 문체를 유지한다
- 수식은 Word 수식 편집기에서 바로 사용 가능한 LaTeX 형태로 제공한다
- 실제 이미지 삽입은 사용자가 직접 수행하며 에이전트는 추천만 한다
- Claude Agent SDK 기반으로 동작한다 (.claude/agents/ 하위 마크다운 파일로 에이전트 정의)
- 외부 API나 별도 서버 없이 로컬에서 실행된다
- pptx 강의안은 사용자가 PDF로 직접 변환하여 제출한다 (에이전트가 pptx를 직접 파싱하지 않음)
- xlsx 파일은 시스템이 제공하는 openpyxl 기반 추출 스크립트로 텍스트 변환 후 처리한다

## 성공 기준

- [ ] 실험 매뉴얼, 강의안(PDF), 레포트 양식, 실험 결과값(xlsx 또는 CSV) 파일을 입력하면 문항별 답변이 생성된다
- [ ] 생성된 답변이 GPTZero 및 Turnitin AI 탐지기에서 AI 작성으로 탐지되지 않는다
- [ ] 표절률이 20% 미만으로 유지된다
- [ ] 족보의 내용이 답변에 포함되지 않는다 (문체만 반영됨)
- [ ] 수식이 Word 수식 편집기에 붙여넣기 가능한 LaTeX 형식으로 출력된다
- [ ] 결과물이 Word(.docx) 파일 또는 즉시 복붙 가능한 형태로 제공된다
- [ ] 각 문항 답변에 삽입할 사진/그림 추천 목록과 출처가 제공된다
- [ ] 참고문헌 목록이 인용 형식에 맞게 생성된다
- [ ] pre-lab과 post-lab 모두 지원한다
- [ ] 열전달, 기계진동, 유체역학, 동역학 4개 분야 모두 지원한다
- [ ] xlsx 실험 데이터가 자동으로 텍스트 추출되어 답변에 반영된다
