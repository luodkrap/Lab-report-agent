# 기계공학실험(2) 레포트 에이전트

## 목적

pre-lab / post-lab 레포트 초안을 자동 생성하는 멀티 에이전트 시스템.

## 사용법

"레포트 작성", "pre-lab", "post-lab" 등을 언급하면 **orchestrator** 에이전트가 자동 실행된다.
실험명·종류(pre/post)·입력 파일 위치를 알려주면 된다.

## 디렉터리

```
input/manual/   — 실험 매뉴얼
input/lecture/  — 강의안
input/form/     — 레포트 양식
input/gpt/      — 족보(선배 레포트) — 문체 분석에만 사용, 내용 참조 금지
output/         — 최종 결과물
tmp/            — 에이전트 간 중간 파일
```

## 핵심 규칙

- 족보는 **문체 분석 전용** (Style Analyzer만 접근). 수치·결론 절대 참조 금지.
- LaTeX 수식은 Word 수식 편집기 호환 형식으로 작성.
- 응답 언어: 한국어.
- **정보 부족 시 선질문**: 레포트 작성에 필요한 정보(담당 문항, 팀원 문항, 분량 조건, 실험 데이터 등)가 부족하면 작성 전에 반드시 사용자에게 확인한다.
- **수식 직접 기재**: "강의안식", "강의안에 나온 식", "이론 수업에서 배운 공식" 등 간접 참조 금지. 수식을 언급할 때는 해당 수식을 LaTeX로 직접 작성한다.
- **문항 간 연계성**: 레포트는 팀 과제이므로, 담당 문항만 독립적으로 쓰지 않고 다른 팀원이 작성하는 문항과의 흐름·연결을 고려한다.
- **분량 조건 필수 준수**: 지정된 분량 조건(글자 수, 페이지 수 등)을 항상 충족해야 한다.
- **AI티 제거 + 보고서 문체 유지**: AI가 쓴 느낌을 최소화하되, 공학 보고서의 격식(~다 체, 객관적 서술, 데이터 기반 논증)은 반드시 유지한다.
